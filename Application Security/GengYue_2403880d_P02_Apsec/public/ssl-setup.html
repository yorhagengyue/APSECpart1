SSL/TLS Configuration Guide

Status: SSL/TLS support is now configured! Use HTTPS for secure connections.

What is SSL/TLS?

SSL (Secure Sockets Layer) and its successor TLS (Transport Layer Security) are cryptographic protocols that provide:

  • Encryption: All data between client and server is encrypted
  • Authentication: Verifies the server's identity
  • Data Integrity: Ensures data hasn't been tampered with

Quick Start - Using HTTPS

  1. Generate SSL Certificate (if not already done): 
    npm run cert:generate
  2. Start HTTPS Server: 
    npm run start:https
  3. Access the Application:
    • HTTPS: https://localhost:5443
    • HTTP: http://localhost:5000 (auto-redirects to HTTPS)

Security Features Enabled

1. HTTP Strict Transport Security (HSTS)

Forces browsers to use HTTPS for all future requests:

Strict-Transport-Security: max-age=31536000; includeSubDomains; preload
2. Content Security Policy (CSP)

Prevents XSS attacks by controlling resource loading:

Content-Security-Policy: default-src 'self' https:; ...
3. Automatic HTTP to HTTPS Redirect

All HTTP requests automatically redirect to HTTPS with 301 status

Browser Security Warning

Browsers will show a security warning for self-signed certificates. This is normal for development.
To proceed in different browsers:
  • Chrome: Click "Advanced" → "Proceed to localhost (unsafe)"
  • Firefox: Click "Advanced" → "Accept the Risk and Continue"
  • Edge: Click "Advanced" → "Continue to localhost (unsafe)"
  • Safari: Click "Show Details" → "visit this website"

Production SSL Certificate

For production, you need a certificate from a Certificate Authority (CA):

Option 1: Let's Encrypt (Free)
  • Use Certbot for automatic certificate generation
  • Automatic renewal every 90 days
  • Trusted by all browsers
Option 2: Commercial SSL Certificate
  • Purchase from providers like DigiCert, Comodo, GeoTrust
  • Extended validation (EV) certificates available
  • Longer validity periods
Option 3: Cloud Provider SSL
  • AWS Certificate Manager (ACM)
  • Azure App Service Certificates
  • Google Cloud SSL Certificates

Certificate Files

The application uses the following certificate files:

  • certificates/server.crt - The SSL certificate
  • certificates/server.key - The private key (keep secure!)
Important: Never commit private keys to version control! The certificates directory is already in .gitignore.
To use your own certificate:
  1. Replace server.crt with your certificate
  2. Replace server.key with your private key
  3. Restart the server

Environment Variables

Configure ports in your .env file:

# HTTP Port (redirects to HTTPS)
PORT=5000

# HTTPS Port
HTTPS_PORT=5443

Testing SSL/TLS

After starting the HTTPS server, test these security features:

  1. HTTPS Redirect: Visit http://localhost:5000 - should redirect to HTTPS
  2. Certificate Info: Click the padlock icon in your browser to view certificate details
  3. Security Headers: Use browser DevTools Network tab to inspect response headers
  4. SSL Labs Test: For production, use SSL Labs to test your configuration
